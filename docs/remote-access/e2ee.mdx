---
title: "End-to-End Encryption (E2EE)"
description: "Securely access your Vibe Kanban instance from any browser with end-to-end encrypted remote access"
---

Vibe Kanban supports end-to-end encrypted remote access, allowing you to securely use the Web UI from any browser to manage tasks running on a remote machine. The gateway server acts as a zero-knowledge message router — **it never has access to your data**. All content is encrypted between your browser and the machine running `vibe-kanban`.

<Info>
E2EE remote access is currently in <Badge color="orange">Beta</Badge>. The gateway can be self-hosted on any infrastructure you control.
</Info>

## Architecture

```
Browser (holds keys)  ←──HTTPS──→  Gateway (zero-knowledge)  ←──WSS──→  Your Machine (vibe-kanban)
```

```
┌──────────────┐                              ┌──────────────────────┐
│  Machine A   │◄── Encrypted messages ──────►│                      │
│  (vibe-kanban│                              │     Gateway          │
│   server)    │                              │     (e2ee-gateway)   │
├──────────────┤                              │                      │
│  Machine B   │◄── Encrypted messages ──────►│  - Serves Web UI     │
│  (vibe-kanban│                              │  - User accounts     │
│   server)    │                              │  - Message routing   │
└──────────────┘                              └──────────┬───────────┘
                                                         │
┌──────────────┐                                         │
│  Browser     │◄── Encrypted messages ─────────────────►┘
│  (Web UI)    │
└──────────────┘
```

**Key properties:**
- The gateway cannot decrypt any content — it only routes encrypted messages
- The gateway serves the complete vibe-kanban Web UI — no local installation needed on the client
- Multiple machines can register to the same gateway
- The browser derives the same encryption keys from a shared master secret
- All operations are scoped per-user — users cannot access each other's machines

## Quick Start

<Steps>
<Step title="Deploy the Gateway">
Deploy the `e2ee-gateway` binary on a server with a public IP or domain. The gateway is a single binary that includes the Web UI.

```bash
GATEWAY_PORT=443 ./e2ee-gateway
```

The first user to sign up automatically becomes the admin. After that, registration is closed.

**Gateway environment variables:**

| Variable | Default | Description |
|----------|---------|-------------|
| `GATEWAY_HOST` | `0.0.0.0` | Bind address |
| `GATEWAY_PORT` | `9090` | Listen port |
| `GATEWAY_DATABASE_URL` | `sqlite:gateway.db?mode=rwc` | SQLite database path |
| `GATEWAY_ALLOWED_ORIGINS` | *(none)* | CORS allowed origins (comma-separated) |
| `GATEWAY_SESSION_TTL_HOURS` | `168` (7 days) | Session token lifetime |
</Step>

<Step title="Connect Your Machine">
On the machine running `vibe-kanban`, authenticate with the gateway:

```bash
vibe-kanban login --gateway https://your-gateway.example.com
```

This will prompt for your email and password, generate a master secret, and save credentials locally. **Save the printed master secret — you'll need it in the browser.**

Then start the server with the gateway URL:

```bash
VK_GATEWAY_URL=https://your-gateway.example.com vibe-kanban
```

The server connects to the gateway automatically.

<Note>
Your device private key is stored locally and never leaves your machine.
</Note>
</Step>

<Step title="Access from Any Browser">
Open `https://your-gateway.example.com` in any browser.

1. **Log in** — on the login page, enter the email and password you registered with
2. **Enter your master secret** — paste the base64 string from the previous step
3. **Select your machine** — you'll see a list of your online machines
4. **Done** — the full vibe-kanban interface loads, all data is end-to-end encrypted

All API calls, WebSocket streams, and file uploads are transparently encrypted in your browser. The gateway only sees encrypted data.
</Step>
</Steps>

## Connection Recovery

If the connection drops (network issues, server restart, etc.), the browser will automatically reconnect with exponential backoff. After 5 failed attempts, you'll be returned to the machine selection screen to reconnect manually.

## Cryptography

The E2EE system uses industry-standard cryptographic primitives:

### Key Derivation

1. **Master Secret**: A 32-byte random secret generated during `vibe-kanban login`
2. **Device Keypairs**: Master Secret is used to derive:
   - Ed25519 keypair for authentication
   - X25519 keypair for content encryption

### Data Encryption

1. **Key Agreement**: X25519 Diffie-Hellman between sender and recipient
2. **Key Derivation**: HKDF to derive a Data Encryption Key (DEK)
3. **Encryption**: XChaCha20-Poly1305 AEAD for authenticated encryption

## CLI Commands

| Command | Description |
|---------|-------------|
| `vibe-kanban login --gateway <url>` | Authenticate with a gateway and register device |
| `vibe-kanban logout` | Remove stored gateway credentials |
| `vibe-kanban status` | Show gateway connection status |

## Security Considerations

<Warning>
- The gateway **never** has access to decrypted content — all encryption happens on your devices
- If you lose your master secret, your pairing cannot be recovered — there is no key escrow
- Always deploy the gateway over HTTPS in production
</Warning>

## Troubleshooting

<AccordionGroup>
<Accordion title="No machines appear after logging in">
  - Check that the `vibe-kanban` server is running with `VK_GATEWAY_URL` set
  - Look at the server logs for connection messages
  - Verify the gateway URL matches on both server and browser
</Accordion>

<Accordion title="Pairing fails / Invalid master secret">
  - Ensure you're pasting the complete base64 master secret (exactly 44 characters for a 32-byte secret)
  - The master secret is printed during `vibe-kanban login` — check your terminal output
</Accordion>

<Accordion title="Connection keeps dropping">
  - The browser automatically reconnects up to 5 times with exponential backoff
  - If it keeps failing, check the gateway server logs and network connectivity
  - Verify the gateway is accessible from both the server and the browser
</Accordion>

<Accordion title="Request timeout">
  - Check that the remote `vibe-kanban` server is running and connected to the gateway
  - Normal requests timeout after 30 seconds, file uploads after 120 seconds
  - Look at the server logs for any error messages
</Accordion>

<Accordion title="Authentication failures">
  - Verify your credentials are correct
  - Try logging out and back in: `vibe-kanban logout` then `vibe-kanban login`
  - Check the gateway logs for error messages
</Accordion>
</AccordionGroup>
